## 💻 Ключевые фрагменты кода

### Функция подсчёта гласных (main.py)

```python
def count_vowels(text: str) -> int:
    vowels = "аеёиоуыэюяАЕЁИОУЫЭЮЯ"
    count = 0
    for char in text:
        if char in vowels:
            count += 1
    return count
```

**Особенности:**
- Типизация через `text: str -> int`
- Поддержка русского алфавита (включая ё)
- Поддержка верхнего и нижнего регистра
- Простая и эффективная реализация

---

### Тесты (test_main.py)

```python
import pytest
from main import count_vowels

def test_only_vowels():
    assert count_vowels("аеёиоуыэюя") == 10
    assert count_vowels("АЕЁИОУЫЭЮЯ") == 10

def test_no_vowels():
    assert count_vowels("бвгджзклмнпрст") == 0
    assert count_vowels("12345 !@#$%") == 0

def test_mixed_case():
    assert count_vowels("Привет, мир!") == 3
    assert count_vowels("ЭТО ТЕСТ") == 3
    assert count_vowels("aEiOu АЕИОУ") == 5
```

**Покрытие сценариев:**
- ✅ Строки только из гласных (нижний и верхний регистр)
- ✅ Строки без гласных (согласные, цифры, спецсимволы)
- ✅ Смешанный регистр с знаками препинания
- ✅ Пустые строки и пробелы

## 🎓 Чему я научилась в этом проекте

### ✅ Pytest
- Установка и настройка pytest
- Написание тестовых функций с префиксом `test_`
- Использование `assert` для проверок
- Запуск тестов с различными флагами (`-v`, `--cov`)

### ✅ Модульное тестирование
- Принцип AAA (Arrange-Act-Assert)
- Покрытие различных сценариев (граничные случаи)
- Тестирование граничных условий (пустые строки, только гласные, без гласных)
- Изоляция тестов друг от друга

### ✅ Type Hints
- Использование аннотаций типов (`text: str -> int`)
- Улучшение читаемости кода
- Поддержка IDE для автодополнения

### ✅ Best Practices
- Именование тестов с понятными именами (`test_only_vowels`)
- Разделение тестов по сценариям
- Использование фикстур (в будущих проектах)
- Следование принципу единственной ответственности

### ✅ Работа с Git
- Структура проекта для тестирования
- Версионирование тестов
- Лицензирование (MIT)
